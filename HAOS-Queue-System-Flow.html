HAOS Queue System

Flow การทำงานทั้งหมดของระบบจัดการคิว (Queue Management System)

.NET 8.0 WebApi Angular 18 WebClient SignalR Real-time PostgreSQL + EF Core อัปเดต: 2026-06-26

1 ภาพรวมระบบ

ระบบ HAOS Queue เป็นระบบจัดการคิวสำหรับเคาน์เตอร์บริการและห้องรอ มีความสามารถหลัก:

  • ออกบัตรคิว ผ่าน Kiosk / Mobile / เจ้าหน้าที่
  • เรียกคิว ที่เคาน์เตอร์ (Service Point) แบบ Real-time
  • ส่งต่อคิว (Forward) ไปยังห้อง/บริการอื่น
  • พักคิว (Hold/Break) พร้อมระบุเหตุผล
  • จอแสดงผล TV แสดงคิวที่กำลังเรียกแบบสด
  • รายงาน & Dashboard วิเคราะห์เวลารอ/เวลาให้บริการ

4 Flow หลัก: วงจรชีวิตของคิว

ตั้งแต่ออกบัตรจนเสร็จสิ้น พร้อม method ที่ถูกเรียกและ event ที่ยิง:

1
สร้างคิว — CreateAsync
  • CounterService.GetNextQueueNo() → สร้างเลขคิว เช่น A-001
  • สร้าง TbQueue + TbQueueService (status = None หรือ CheckIn ถ้า AutoCheckin)
SignalR: TriggerQueueChange
2
เช็คอิน — CheckinAsync [ข้ามได้ถ้า AutoCheckin]
  • ValidateCheckinAsync() ตรวจสอบสถานะก่อน
  • status = CheckIn, ตั้ง CheckinTime
SignalR: TriggerQueueChange
3
เรียกคิว — NextAsync (ที่เคาน์เตอร์)
  • เลือกคิวเอง หรือ auto FIFO จากคิว CheckIn ที่เก่าสุด
  • status = Inprogress, ตั้ง StartTime + PointId
  • สร้าง TbQueueCalling (บันทึกการเรียก)
SignalR: TriggerQueueChange + TriggerQueueCalling
4
ระหว่างให้บริการ — เลือกได้หลายทาง
A · เสร็จสิ้นCompleteAsync: status = Completed, ตั้ง EndTime, ลบ TbQueueCalling
B · ส่งต่อForwardAsync: ปิด service เดิม (Completed) + สร้าง service ใหม่ (None) → กลับ step 2
C · พักคิวPauseAsync: status = Holding, บันทึก TbQueuePauseHistory, ลบ TbQueueCalling
D · เรียกซ้ำRecallAsync: ยิง event ซ้ำ ไม่เปลี่ยนสถานะ

7 Flow การเรียกคิว

POST /api/queue/next → NextAsync(QueueNextInputModel)  |  Input: PointId, RoomId, ServiceTypeId, QueueId (optional)

1
ตรวจสอบการเรียกซ้อน — เคาน์เตอร์นี้กำลังเรียกคิวอื่นอยู่ไหม? ถ้ามี → error INVALID_QUEUE_NO_DUPLICATE
2
เลือกคิวเป้าหมาย
  • ระบุ QueueId → เรียกคิวนั้น (ต้องสถานะ None / CheckIn / Holding)
  • ไม่ระบุ → auto เลือก CheckIn ที่เก่าสุด (FIFO)
  • ไม่พบ → error INVALID_QUEUE_NO_WAITING
3
อัปเดต TbQueueService → Inprogress, ตั้ง PointId + StartTime
4
สร้าง TbQueueCalling (QueueNumber, PointId, CallerBy, CallTime) — unique index กัน (QueueId, PointId) ซ้ำ SignalR: TriggerQueueChange + TriggerQueueCalling
ฝั่ง UI (เจ้าหน้าที่): QueueCallingComponent (/queue/calling) → เลือกเคาน์เตอร์ → กด "เรียกคิวถัดไป" callNextQueue() → คิวแสดงในวิดเจ็ตด้านบน (กระพริบ 10 วินาที)

12 Flow Authentication & Authorization

12.1 Login Flow  POST /api/auth/sign-in

1
ส่ง credential — username / password + RememberMe
2
ตรวจสอบ — VerifyPassword (bcrypt) · นับ FailedLoginCount เกิน MaxFailedLoginCount → ล็อกบัญชี · ตรวจ User.Status & UserGroup.Status
3
ออก Token
  • Access Token (JWT, HS256) — claims: UserId, Username, Email, fullName, roles(UserGroupId), remember · อายุ TokenLifetimeHours
  • Refresh Token (32 bytes base64) เก็บใน TbUserRefreshToken · อายุ Token + 12 ชม.
4
คืนค่า — AccessToken, RefreshToken, FromFirstLogin (ForceChangePassword), UserInfo (permissions)

12.2 Refresh Token  POST /api/auth/refresh-token

ส่ง RefreshToken เดิม
ตรวจมีอยู่ + ยังไม่หมดอายุ→
ออก AccessToken + RefreshToken ใหม่ตัวเก่าใช้ไม่ได้

12.3 Authorization (PermissionAuthorize)

JWT claim roles→ UserGroupId
