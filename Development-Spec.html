A · ต่อยอดระบบคิวเดิม

ส่วนที่พัฒนาบนฐานเดิม (Queue Engine + SignalR + Report) ทำได้โดยไม่ต้องรอ HIS
A1🔀 Service Routing Engine 3.3 / routingเฟส 1+2P:กลาง
รายละเอียด
ตั้งกฎ "เสร็จจุดนี้ → ไปจุดไหนต่อ" 3 โหมด: Always / เงื่อนไข HIS / Manual · เมื่อกด "เสร็จ" Engine ประเมินกฎ + flag จาก HIS → สร้าง service ถัดไปอัตโนมัติ (ต่อยอดจาก Forward) · กันลูปไม่รู้จบ · ดูเอกสาร Service Routing ฉบับเต็ม
Backend
CRUD /api/service-route POST /api/queue/complete (ผูก engine) · Entity: TbServiceRoute TbQueueOrderFlag · เพิ่มฟิลด์ RouteId, Sequence ใน TbQueueService
Frontend
หน้า "ตั้งค่าเส้นทางบริการ" (ตารางกฎ + preview ผัง) · ปรับหน้า Calling ให้แสดงปลายทางอัตโนมัติ/ยืนยัน
ตรวจรับ
ตั้งกฎ "ซักประวัติ→ห้องตรวจ" แล้วกดเสร็จ คิวเข้าห้องตรวจอัตโนมัติ
กฎเงื่อนไข DRUG_ORDER ทำงานเฉพาะเมื่อมี flag · ป้องกันลูปได้
A2⚡ Priority / Triage (คิวลัด) 3.3.2 / 3.3.10เฟส 1P:กลาง
รายละเอียด
เพิ่มระดับความเร่งด่วนให้คิว → เรียงคิวลัดอัตโนมัติ · รองรับ Emergency Triage Score สำหรับห้องฉุกเฉิน · จัดลำดับการเรียกตาม priority + เวลา
Backend
เพิ่ม PriorityLevel, TriageScore ใน TbQueueService · ปรับ logic NextAsync ให้เรียงตาม priority ก่อน FIFO · PUT /api/queue/priority
Frontend
แสดงป้ายความเร่งด่วน (สี) ในรายการคิว/จอ TV · ปุ่มปรับระดับ triage
ตรวจรับ
คิวเร่งด่วนถูกเรียกก่อนคิวปกติที่มาก่อน
แสดงสีความเร่งด่วนบนจอ TV ได้
A3🎫 บัตรคิว / ใบนำทาง (ปรับแต่งได้) 3.1.6 / 3.3.6เฟส 1P:ต่ำ
รายละเอียด
พิมพ์บัตรคิว (เลขคิว, ข้อมูลทั่วไป, QR) + ใบนำทาง (Navigation slip) ตาม template ที่ admin ปรับแต่งได้ · ต่อยอดจาก report engine (Stimulsoft) เดิม
Backend
POST /api/form/print-ticket /print-navigation · ใช้ ReportService เดิม + template .mrt · TbTicketTemplate (config)
Frontend
หน้าตั้งค่า template บัตร · trigger พิมพ์จาก KIOSK/หน้า Calling
ตรวจรับ
พิมพ์บัตรมี QR ที่สแกนติดตามได้ · admin แก้ layout แล้วมีผล
A4📺 จอแสดงผล & Dashboard (ปรับปรุง) 3.4เฟส 1P:ต่ำ
รายละเอียด
เพิ่ม: แถบสีสถานะตามแผนก, แสดงความเร่งด่วน, สถานะเฉพาะห้องยา/การเงิน (กำลังจัดยา/รอการเงิน), ปรับ layout ต่อแผนกได้ · Dashboard ภาพรวมพร้อมเกณฑ์สีเวลารอ (มีฐานแล้ว)
Backend
เพิ่มสถานะย่อย (sub-status) ใน enum สถานะคิว · GET /api/queue/get-dashboard (ขยาย)
Frontend
QueueTV: เพิ่มธีม/แถบสีต่อแผนก · Dashboard: การ์ดสถิติ + สีเตือน
ตรวจรับ
จอแสดงคิวกำลังรอ/รักษา/เรียกข้าม + ชื่อ + เวลารอ ตาม TOR 3.4.2–3.4.3
A5🗂️ Master Data Mapping supportเฟส 1P:ต่ำ
รายละเอียด
เพิ่มฟิลด์จับคู่ master ระบบคิว ↔ HIS (clinic/service/payer code) ทำเป็น config UI · เตรียมฟิลด์ HN/VN
Backend
เพิ่ม HisClinicCode ใน TbmRoom, HisServiceCode ใน TbmServiceType, HN, VN, AppointmentId ใน TbQueue/TbQueueService · TbOrderTypeMap (order→จุด)
Frontend
เพิ่มช่อง map ในหน้าตั้งค่าห้อง/บริการ · หน้า Order Type Map
ตรวจรับ
map clinic code ได้ และ routing ใช้ map นี้หาปลายทางถูกต้อง

B · ตู้ KIOSK & Check-in

หน้าจอบนฮาร์ดแวร์ + เชื่อมอุปกรณ์ (card reader / printer / camera)
B1📟 KIOSK กดบัตรคิว 3.10เฟส 1P:กลาง
รายละเอียด
หน้าจอ touch สำหรับผู้ป่วยกดรับบัตร: เสียบบัตรประชาชน → อ่าน CID/ชื่อ/รูป (ในเครื่อง) → เลือกบริการ/สิทธิ์ → พิมพ์บัตร · รองรับโหมดไม่มีบัตร (กดเลือกบริการอย่างเดียว)
Backend
ใช้ POST /api/queue/create (รับ HN/CID/priority) · device token แยก
Frontend
แอป Kiosk (Angular kiosk mode) · integrate SDK: ID card reader, thermal printer 80mm, กล้อง HD
เชื่อมต่อ
เครื่องอ่านบัตร ปชช. (SDK), เครื่องพิมพ์ความร้อน, กล้อง
ตรวจรับ
เสียบบัตร → อ่านชื่อ+รูปได้ → ออกบัตรคิว + พิมพ์อัตโนมัติ
B2📥 ตู้ Check-in 3.11 / 3.6.1เฟส 1P:ต่ำ
รายละเอียด
ผู้ป่วยสแกน QR/บัตร เพื่อยืนยันมาถึง → เช็คอินเข้าคิว · แสดงลำดับคิว
Backend
ใช้ POST /api/queue/checkin GET /get-queue-scancheckin (มีแล้ว)
Frontend
แอป Check-in (touch) + เครื่องอ่านบัตร/QR scanner
ตรวจรับ
สแกนแล้วสถานะเป็น CheckIn + แสดงลำดับคิวที่รอ

C · เชื่อมต่อ HIS

ดูรายละเอียด contract เต็มในเอกสาร HIS Integration Design
C1🛡️ API Gateway + Connector 3.1.3เฟส 2P:สูง
รายละเอียด
ชั้นกลางคุย HIS: OAuth2/mTLS, retry, timeout, circuit breaker, idempotency, audit · HIS Connector service ใน Business layer
Backend
IHisConnector service + config endpoint mapping · webhook receiver base
ตรวจรับ
เรียก HIS ผ่าน Gateway ได้ + มี audit log + จัดการ timeout/retry
C2👤 Patient / Visit 3.1.4 / 3.2เฟส 2P:สูง
รายละเอียด
ค้นผู้ป่วยจาก CID→HN, เปิด Visit (หรือ reverse-flow รับ VN), ลงทะเบียนผู้ป่วยใหม่ (ผ่าน HIS), เปิด Visit หลายคลินิกจากนัด
Backend
เรียก HIS: GET /patients?cid= POST /visits · webhook POST /queue/webhooks/visit-opened · เก็บ HN/VN ในคิว
Frontend
หน้า KIOSK/เคาน์เตอร์: แสดงข้อมูลผู้ป่วย, เลือกคลินิก, เปิด Visit
ตรวจรับ
เสียบบัตร → ดึง HN จาก HIS → เปิด Visit → คิวผูกกับ VN
C3🆔 Eligibility / สปสช. 3.1.2 / 3.1.5เฟส 2P:สูง
รายละเอียด
ตรวจสอบสิทธิ์ สปสช. ผ่านบัตรประชาชน + แสดงสิทธิ์ทั้งหมดให้เลือก (บัตรทอง/ข้าราชการ/ชำระเอง)
Backend
POST /his/eligibility/verify หรือผ่าน NHSO โดยตรง · เก็บ EligibilityCode
Frontend
หน้าเลือกสิทธิ์ใน KIOSK/Check-in
ตรวจรับ
แสดงสิทธิ์ที่ใช้ได้วันนั้น + ผู้ป่วยเลือกได้ + บันทึกกับ Visit
C4📦 Order Signals (ขับ Routing) 3.3.7 / routingเฟส 2P:กลาง
รายละเอียด
รับสัญญาณว่าแพทย์สั่งยา/Lab/X-ray → ตั้ง flag → Routing เพิ่มจุดถัดไป · รับสถานะผล Lab/X-ray เพื่อล็อค/ปลดล็อคคิว
Backend
webhook POST /queue/webhooks/order-created /order-status หรือ pull GET /his/visits/{vn}/orders · TbQueueOrderFlag
ตรวจรับ
HIS แจ้ง DRUG → กดเสร็จห้องตรวจ → คิวไปการเงิน+ห้องจ่ายยาอัตโนมัติ
C5🔗 Deep-link ข้อมูลคลินิก/เภสัช 3.3.9เฟส 2/4P:ต่ำ
รายละเอียด
ปุ่มเปิดหน้า HIS ของ Visit (ส่ง VN) เพื่อดูประวัติยา/แพ้ยา/Vital/Note — แทนการดึงข้อมูลอ่อนไหวเข้าคิว (ลดงาน/ลด PDPA)
Backend
สร้าง deep-link URL + SSO token (ถ้ามี)
Frontend
ปุ่ม "ดูข้อมูลใน HIS" ในหน้าเรียกคิวห้องยา
ตรวจรับ
กดแล้วเปิดหน้า HIS ของผู้ป่วยที่ถูก visit ได้ทันที

D · นัดหมาย

โมดูลใหม่ทั้งหมด — ฝั่งเจ้าหน้าที่และผู้ป่วยออนไลน์
D1📅 นัดหมาย (เจ้าหน้าที่) 3.5.1เฟส 3P:กลาง

Backend
TbAppointment · CRUD /api/appointment · webhook /queue/webhooks/appointment-created
Frontend
หน้าจัดการนัดหมาย (ปฏิทิน/รายการ) สำหรับเจ้าหน้าที่
ตรวจรับ
สร้างนัด → เมื่อถึงวัน คิว pre-load + เช็คอินได้
D2🌐 นัดออนไลน์ (ผู้ป่วย) 3.5.2เฟส 3P:กลาง
รายละเอียด

Backend
/api/public/appointment (auth ผู้ป่วย) · ตรวจสิทธิ์กับ HIS
Frontend
Portal/LIFF app จองออนไลน์ (responsive)

Backend
GET /his/.../refillable-drugs · TbRefillRequest · reuse payment + dispensing
Frontend
Portal/LIFF เติมยา
ตรวจรับ
ขอเติมยา → ชำระ → ส่งคิวจัดยา → แจ้ง LINE + Pickup Code · ติดตามสถานะ 4 ระดับ
